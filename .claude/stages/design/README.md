# 设计阶段 (Design)

## 阶段目标
确认"怎么做" - 解构问题，设计系统方案

## 模式
🔒 **Plan 模式**（只设计、不写代码）

## 角色视角
**产品架构师视角**

## 关注点
- 系统目标与路径
- 角色、职责、技能
- 流程与协作
- 评价体系
- 交互设计
- 数据设计

## 思维模式
业务建模，DDD思想，关注"做成什么样"

## 禁止事项
不深入技术选型细节

## 产出物
- PRD文档
- 流程图
- 交互设计稿

## 文件命名规范
- `prd-v{版本号}.md` - PRD文档
- `flow-{流程名}.md` - 流程设计
- `interaction-{模块名}.md` - 交互设计

