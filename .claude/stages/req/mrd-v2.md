## 1. 用户画像

| 用户类型 | 特征 | 核心诉求 |
|----------|------|----------|
| 货源商/分销商 | 有稳定货源，需铺货到多个店铺 | 快速复制商品到多店 |
| 新手卖家 | 刚开店，想参考爆款快速起步 | 低门槛复制成功商品 |
| 代运营公司 | 管理多个店铺，需高效上架 | 批量处理，提高人效 |

**共同特征：**
- 有批量上架需求（日均 50+ 商品）
- 希望减少重复劳动
- 能接受命令行工具

---

## 2. 用户痛点

### 核心痛点：手动上架效率极低

- 上架一个商品需要 30-60 分钟
- 日均 50 个商品 = 25+ 小时工作量（远超正常工作时间）
- 重复性劳动（复制标题、下载图片、填写SKU...）

### 现有方案的不足

| 方案 | 问题 |
|------|------|
| 手动上架 | 太慢，无法规模化 |
| 现有工具（甩手等） | 需要付费、功能复杂、有学习成本 |
| 官方API | 门槛高、限制多、需要开发能力 |

---

## 3. 使用场景

### 典型场景

1. **爆款复制**：卖家看到一个爆款商品，想快速上架到自己店铺
2. **多店铺货**：货源商需要把商品同步到多个分销店铺
3. **批量上新**：代运营需要批量处理多个品牌的商品上架

### 用户期望的操作流程

```
复制淘宝商品链接 → 粘贴到工具 → 预览确认 → 一键上架
```

**核心体验：** 看到什么商品，就能复制到自己店铺

---

## 4. 市场机会

### 市场规模

- 天猫/淘宝卖家数量：数百万级
- 有批量上架需求的卖家：预估 10%+

### 竞品分析（用户体验角度）

| 竞品 | 用户痛点 |
|------|----------|
| 甩手工具箱 | 功能多但复杂，上手成本高 |
| 大淘营 | 需要付费，中小卖家有顾虑 |
| 淘精灵 | 界面老旧，体验一般 |
| 店盈通 | 功能臃肿，学习曲线陡 |

**机会点：** 做一个简单、轻量、上手即用的工具

---

## 5. MVP 功能范围

### 做什么（用户价值）

| 功能 | 用户价值 |
|------|----------|
| 商品信息采集 | 自动获取标题、图片、价格、SKU，无需手动复制 |
| 本地预览/编辑 | 上架前可修改，避免错误，保留控制权 |
| 一键上架 | 自动完成上架流程，节省 90% 时间 |

### 不做什么（本期不满足）

| 功能 | 原因 |
|------|------|
| 批量采集 | MVP 先验证单品流程 |
| 图片处理（去水印等） | 增加复杂度，后续迭代 |
| 自动改价 | 非核心需求 |
| 多平台支持 | 先聚焦天猫 |

---

## 6. 成功标准（用户视角）

| 指标 | 目标 | 说明 |
|------|------|------|
| 单品上架时间 | < 5 分钟 | 对比手动 30-60 分钟 |
| 时间节省 | > 80% | 核心价值主张 |
| 上架成功率 | > 99% | 工具可靠性 |
| 用户愿意付费 | 验证 PMF | 产品市场匹配度 |
