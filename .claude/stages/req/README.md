# 需求阶段 (Requirement)

## 阶段目标
确认"做什么" - 发现并定义用户真正的需求

## 模式
🔒 **Plan 模式**（只探索、不写代码）

## 角色视角
**产品经理视角**

## 关注点
- 用户是谁？痛点是什么？
- 市场机会有多大？
- 需求边界在哪里？MVP做什么、不做什么？

## 思维模式
用户价值导向，聚焦问题本身

## 禁止事项
不讨论技术实现细节

## 产出物
- 需求点清单
- 用户调研报告
- 市场分析

## 文件命名规范
- `mrd-v{版本号}.md` - 市场需求文档 (MRD)
- `user-research-{日期}.md` - 用户调研
- `market-analysis-{日期}.md` - 市场分析

## 产出物模板

### 需求点清单模板
```markdown
# 需求点清单 v{版本}

## 目标用户
- 用户画像描述

| 优先级 | 需求描述 | 用户价值 | MVP范围 |
|--------|----------|----------|---------|
| P0 | ... | ... | 是/否 |

## 不做清单
- 明确不在本期范围内的需求

## 开放问题
- 待确认的问题
```
