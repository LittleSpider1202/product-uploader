# 淘宝商品一键上架工具 - 需求文档 v1

## 1. 需求概述

| 项目 | 内容 |
|------|------|
| **用户需求** | 输入淘宝商品链接，1:1 复制商品到自己的天猫店铺 |
| **目标用户** | 天猫店卖家 |
| **交互方式** | 命令行工具 (CLI) |
| **上架模式** | 半自动（采集 → 预览编辑 → 确认上架） |
| **技术路线** | 模拟登录（不使用官方 API） |

---

## 2. MVP 功能范围

### 做什么
- [x] 基础信息采集（标题、主图、价格、描述）
- [x] 规格/SKU 采集（颜色、尺码等多规格）
- [x] 详情页图片采集
- [x] 本地预览/编辑
- [x] 模拟登录上架

### 不做什么
- [ ] 图片处理（去水印、添加水印）
- [ ] 批量采集
- [ ] 价格公式/自动改价
- [ ] 多平台支持

---

## 3. 工作流程

```
输入商品链接
    ↓
采集商品信息（标题/图片/价格/SKU/描述）
    ↓
保存到本地（JSON + 图片）
    ↓
预览/编辑（可选修改标题、价格等）
    ↓
模拟登录天猫卖家后台
    ↓
自动填写发布表单
    ↓
确认上架
```

---

## 4. 技术路线

### 采集端
- 爬虫/模拟请求采集淘宝商品页
- 解析商品数据（标题、价格、SKU、描述）
- 下载商品图片到本地

### 上架端
- 模拟登录天猫卖家中心
- 自动化填写商品发布表单
- 处理验证码/二次验证

### 潜在风险
| 风险 | 说明 | 应对 |
|------|------|------|
| 反爬机制 | 淘宝有严格的反爬策略 | 控制频率、模拟真实行为 |
| 图片溯源 | 淘宝检测图片来源 | 可能需要图片处理 |
| 账号风险 | 模拟登录可能触发安全机制 | 使用 Cookie/扫码登录 |
| 验证码 | 频繁操作触发验证 | 人工介入或打码服务 |
