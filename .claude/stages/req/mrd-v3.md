## 1. 用户画像

| 用户类型 | 特征 | 核心诉求 |
|----------|------|----------|
| 天猫卖家 | 有品牌授权，需合规上架 | 快速上架、自动填充合规信息 |
| 货源商/分销商 | 有稳定货源，需铺货到多个店铺 | 快速复制商品到多店 |
| 代运营公司 | 管理多个店铺，需高效上架 | 批量处理，提高人效 |

**共同特征：**
- 有批量上架需求（日均 10+ 商品）
- 商品信息分散在多处（淘宝、表格、证书等）
- 希望减少重复劳动，提高准确性
- 接受 Web 工具操作

---

## 2. 用户痛点

### 核心痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| 上架耗时 | 单品手动上架 30-60 分钟 | 效率低，无法规模化 |
| 信息散落 | 商品信息在淘宝、表格、证书多处 | 来回查找，容易出错 |
| 重复填写 | 厂家信息、物流模板每次都要填 | 重复劳动，浪费时间 |
| 图片管理 | 图片需先传素材库再引用 | 操作繁琐，效率低 |
| 合规要求 | 有机码、生产许可证等必填 | 容易遗漏，审核不过 |

### 现有方案的不足

| 方案 | 问题 |
|------|------|
| 手动上架 | 太慢，无法规模化 |
| 现有工具（甩手等） | 功能复杂、付费、无法对接自有数据 |
| 官方 API | 门槛高、限制多、需要开发能力 |

**市面工具复杂的根本原因：**

淘宝商品上架是**动态表单**机制——不同类目有不同的字段要求，且存在多步骤复杂交互（选类目 → 填基础信息 → 填 SKU → 填物流 → ...）。市面工具为了兼容所有类目，不得不把所有可能的字段都暴露出来，导致界面臃肿、学习成本高。

## 3. 使用场景

### 典型场景

1. **爆款复制**：看到淘宝爆款，快速复制到自己天猫店
2. **品牌铺货**：品牌方把商品同步到多个经销商店铺
3. **合规上架**：有机食品卖家，需自动填充证书信息

### 用户期望的操作流程

```
粘贴链接 → 采集预览 → 关联数据 → 确认字段 → 上架
    ↓           ↓           ↓           ↓
 1 秒完成   SKU/参数    自动匹配     图片自动传
            一览无余    编码信息     素材库
```

**核心体验：**
- 采集后即可预览完整信息
- 数据源自动匹配，无需手动查找
- 图片自动上传到千牛素材库
- 模板记住常用配置，下次自动应用

## 4. 市场机会

### 市场规模

- 天猫/淘宝卖家数量：数百万级
- 有批量上架需求的卖家：预估 10%+
- 有合规信息管理需求：食品、保健品等类目

### 竞品分析（用户体验角度）

| 竞品 | 用户痛点 |
|------|----------|
| 甩手工具箱 | 功能多但复杂，无法对接自有数据源 |
| 大淘营 | 需要付费，中小卖家有顾虑 |
| 淘精灵 | 界面老旧，体验一般 |
| 店盈通 | 功能臃肿，学习曲线陡 |

**差异化机会：**
- 支持对接钉钉/飞书表格作为数据源
- 自动匹配商品与合规信息
- 类目模板记忆常用配置
- 图片自动上传到千牛素材库

## 5. MVP 功能范围

### 做什么（用户价值）

| 功能模块 | 用户价值 |
|----------|----------|
| **商品采集** | 粘贴链接自动采集，预览 SKU、参数、图片 |
| **数据源对接** | 关联钉钉表格，自动匹配商家编码、有机码等 |
| **类目模板** | 记住发货地、物流模板等常用配置 |
| **素材库上传** | 图片自动上传到千牛素材库指定目录 |
| **价格策略** | 支持统一加价策略（如上调 10%） |
| **一键上架** | 自动完成天猫发布流程 |

### 5 步上架流程

| 步骤 | 名称 | 用户操作 |
|------|------|----------|
| Step 1 | 采集商品 | 粘贴链接、选择数据源、选择模板 |
| Step 2 | 选择类目 | 确认天猫发布类目 |
| Step 3 | 确认字段 | 检查必填项、上传图片到素材库 |
| Step 4 | 编辑 SKU | 设置价格策略、库存 |
| Step 5 | 提交上架 | 选择上架方式、确认提交 |

### 不做什么（本期不满足）

| 功能 | 原因 |
|------|------|
| 批量采集 | MVP 先验证单品流程 |
| 图片处理（去水印、AI 生成） | 增加复杂度，后续迭代 |
| 多平台支持 | 先聚焦天猫 |
| 跨店铺同步 | 后续迭代 |

## 6. 成功标准（用户视角）

| 指标 | 目标 | 说明 |
|------|------|------|
| 单品上架时间 | < 5 分钟 | 对比手动 30-60 分钟 |
| 时间节省 | > 80% | 核心价值主张 |
| 上架成功率 | > 95% | 工具可靠性 |
| 数据匹配率 | > 90% | 数据源自动匹配准确率 |
| 用户愿意付费 | 验证 PMF | 产品市场匹配度 |

### 验证方式

- 内部试用：先用自己店铺验证流程
- 小范围测试：邀请 5-10 个目标用户试用
- 收集反馈：上架成功率、耗时、体验问题
